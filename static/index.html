零配置启动

服务不需要 .env。容器内固定监听 0.0.0.0:55555，对外访问端口由 Docker 端口映射决定。

首次启动会使用内置默认值，渠道配置写入 data/channels.json，API Key 写入 data/api_keys.json，系统设置写入 data/settings.json。只要持久化 data/ 目录，渠道、API Key、统计和设置都会随容器重建保留。

固定监听所有网卡，便于 Docker 映射到宿主机端口。

宿主机端口请在 Docker 中映射，例如 8000:55555。

请求处理

这些设置影响所有代理入口：OpenAI Chat Completions、OpenAI Responses 和 Anthropic Messages。

普通请求和流式请求都会使用同一套超时配置。请求体限制在进入代理前检查，适合防止意外上传超大 payload。

单位：秒，超过此时间未响应的请求将被终止

单位：MB，超过此大小的请求将被拒绝

负载均衡

渠道按优先级分组，再在同组内按权重轮询；故障渠道会暂时避开，冷却后再尝试恢复。

这里是全局健康策略。具体模型、渠道权重、优先级和启停状态在「渠道管理」与「模型分组」中维护。

渠道连续请求失败达到此次数后将被暂时剔除

单位：秒，剔除的渠道经过此时间后自动恢复探测

日志

控制服务端输出级别。配置会写入 settings.json，重启后生效。

日志级别 需重启

DEBUG 会输出更多代理细节；生产环境通常使用 INFO 或 WARNING。

统计时区 热更新

用于决定日聚合切日和"今日"边界。IANA 时区名（参考 wikipedia.org/wiki/List_of_tz_database_time_zones）。留空则使用服务器系统时区。

请求记录数据库

每次请求都会自动记录基础统计（渠道、模型、耗时、Token 用量、状态码等），无需任何配置。

如需记录完整的消息内容（Header、Body），在下方「原始信息保存」中按需开启。这部分数据体积大，默认关闭。

首页图表使用的聚合统计单独保存在 data/stats.db，请求记录默认保存在 data/request_logs.db，不受此处设置影响。

数据库类型

SQLite 适合单机部署，开箱即用；PostgreSQL 适合多实例共享或需要外部访问的场景。切换后立即生效，已有记录不会迁移。

SQLite 文件路径

路径固定，不可修改。如需备份或迁移，直接复制此文件即可。

PostgreSQL 连接串 热更新

格式：postgresql://用户名:密码@主机:端口/数据库名。出于安全考虑，页面只展示脱敏后的值，修改时需重新输入完整连接串。

原始信息保存

系统默认只记录基础统计，不保存消息内容。下方四项按需开启：

  • 请求 Header：客户端发来的 HTTP 头，含鉴权 Key、客户端类型等，可用于追踪调用来源。
  • 返回 Header：上游 API 返回的 HTTP 头，含速率限制剩余次数等调试信息。
  • 请求 Body：完整的消息列表（messages）及所有参数，体积最大，审计时最有价值。
  • 返回 Body：上游返回的完整响应，包含模型输出的所有内容。
Body 截断限制 热更新

单位：KB。仅在开启 Body 保存时生效。

当请求或响应的 Body 超过此大小时，数据库中不会存储完整内容，而是替换为一个摘要对象，例如：

{"_truncated": true, "_original_bytes": 126400, "_limit_bytes": 65536, "_preview": "消息开头内容..."}

这样可以防止长对话（大 context）消耗过多存储空间。0 表示不限制，完整保存所有内容，默认 64 KB。

数据保留策略 热更新

服务启动后 10 秒自动执行一次，之后每 24 小时运行一次。0 表示不清理，保留所有历史数据。

IP 白名单

控制哪些 IP 可以访问任意路径（管理接口与代理接口均受保护）。白名单为空时，不做任何限制。

规则格式说明

每行一条规则，用 , 分隔 4 列。# 开头的行为注释，空行忽略。

多行之间是「或」关系：请求命中任意一行即放行。同一行内路径、方法、IP 三者同时满足才算命中。

列 示例 说明
path_pattern /admin/* 路径，支持 * 通配符；/admin/* 覆盖管理接口，/v1/* 覆盖代理接口
methods * 或 GET|POST * 表示不限方法；多个方法用竖线 | 分隔
ip_cidr 192.168.1.0/24 精确 IP 或 CIDR 网段，支持 IPv4 和 IPv6
description 家庭内网 备注说明，会出现在 403 错误信息中 
# 注释：以 # 开头的行会被忽略
path_pattern,methods,ip_cidr,description
/admin/*,*,10.1.1.0/24,家庭内网
/admin/*,*,127.0.0.1,本机
/admin/stats,GET,203.0.113.5,公司只读
注意：系统当前使用直连 IP（TCP 连接地址）进行匹配。若服务部署在 nginx/caddy 等反向代理后面，所有请求的来源 IP 都会是代理本身（通常 127.0.0.1），白名单将以代理 IP 为准。请在代理层做访问控制，或等后续版本支持 X-Forwarded-For 后再在此配置。
