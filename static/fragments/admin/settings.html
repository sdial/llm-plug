零配置启动

服务不需要 .env。容器内固定监听 0.0.0.0:55555，对外访问端口由 Docker 端口映射决定。

首次启动会使用内置默认值，渠道配置写入 data/channels.json，API Key 写入 data/api_keys.json，系统设置写入 data/settings.json。只要持久化 data/ 目录，渠道、API Key、统计和设置都会随容器重建保留。

固定监听所有网卡，便于 Docker 映射到宿主机端口。

宿主机端口请在 Docker 中映射，例如 8000:55555。

请求记录数据库

每次请求都会自动记录基础统计（渠道、模型、耗时、Token 用量、状态码等），无需任何配置。

如需记录完整的消息内容（Header、Body），在下方「原始信息保存」中按需开启。这部分数据体积大，默认关闭。

首页图表使用的聚合统计单独保存在 data/stats.db，请求记录默认保存在 data/request_logs.db，不受此处设置影响。

SQLite 文件路径

请求记录仅使用 SQLite3，本地文件路径固定。如需备份或迁移，直接复制此文件即可。

原始信息保存

系统默认只记录基础统计，不保存消息内容。下方四项按需开启：

  • 请求 Header：客户端发来的 HTTP 头，含鉴权 Key、客户端类型等，可用于追踪调用来源。
  • 返回 Header：上游 API 返回的 HTTP 头，含速率限制剩余次数等调试信息。
  • 请求 Body：完整的消息列表（messages）及所有参数，体积最大，审计时最有价值。
  • 返回 Body：上游返回的完整响应，包含模型输出的所有内容。
Body 截断限制 热更新

单位：KB。仅在开启 Body 保存时生效。

当请求或响应的 Body 超过此大小时，数据库中不会存储完整内容，而是替换为一个摘要对象，例如：

{"_truncated": true, "_original_bytes": 126400, "_limit_bytes": 65536, "_preview": "消息开头内容..."}

这样可以防止长对话（大 context）消耗过多存储空间。0 表示不限制，完整保存所有内容，默认 64 KB。
