IP 白名单

控制哪些 IP 可以访问任意路径（管理接口与代理接口均受保护）。白名单为空时，不做任何限制。

规则格式说明

每行一条规则，用 , 分隔 4 列。# 开头的行为注释，空行忽略。

多行之间是「或」关系：请求命中任意一行即放行。同一行内路径、方法、IP 三者同时满足才算命中。

列 示例 说明
path_pattern /admin/* 路径，支持 * 通配符；/admin/* 覆盖管理接口，/v1/* 覆盖代理接口
methods * 或 GET|POST * 表示不限方法；多个方法用竖线 | 分隔
ip_cidr 192.168.1.0/24 精确 IP 或 CIDR 网段，支持 IPv4 和 IPv6
description 家庭内网 备注说明，会出现在 403 错误信息中 
# 注释：以 # 开头的行会被忽略
path_pattern,methods,ip_cidr,description
/admin/*,*,10.1.1.0/24,家庭内网
/admin/*,*,127.0.0.1,本机
/admin/stats,GET,203.0.113.5,公司只读
注意：系统当前使用直连 IP（TCP 连接地址）进行匹配。若服务部署在 nginx/caddy 等反向代理后面，所有请求的来源 IP 都会是代理本身（通常 127.0.0.1），白名单将以代理 IP 为准。请在代理层做访问控制，或等后续版本支持 X-Forwarded-For 后再在此配置。
