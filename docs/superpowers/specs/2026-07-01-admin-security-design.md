# 管理员安全功能设计

## 概述

为管理后台增加密码修改功能和阶梯式IP封锁机制，提升安全性。

## 需求

1. **修改密码**：设置页和登录页两处入口
2. **阶梯式IP封锁**：超过尝试次数后，封锁时间递增
3. **强制删除密码**：登录页显示忘记密码时的重置方法

## 设计方案

### 1. 后端API

#### 新增端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/admin/auth/change-password` | POST | 修改密码，需登录+CSRF |
| `/admin/auth/security-config` | GET | 获取安全配置 |
| `/admin/auth/security-config` | PUT | 更新安全配置，需登录+CSRF |

#### 修改密码请求体

```json
{
  "old_password": "旧密码",
  "new_password": "新密码",
  "confirm_password": "确认新密码"
}
```

#### 修改密码流程

1. 验证CSRF Token
2. 验证旧密码
3. 检查新密码 == 确认密码
4. 检查新密码长度 >= 6位
5. 更新 `admin_auth.json` 的 `password_hash`
6. 撤销所有现有会话（强制重新登录）

### 2. IP阶梯封锁机制

#### 封锁阶梯表

| 失败次数 | 封锁时间 |
|---------|---------|
| 1-10 | 60秒 |
| 11-20 | 120秒 |
| 21-30 | 240秒 |
| 31-40 | 600秒 |
| 41-50 | 3600秒 |
| 51-60 | 86400秒 |
| 60+ | 86400秒（封顶） |

#### 数据结构（内存）

```python
_login_attempts: dict[str, list[float]] = {}  # IP -> [时间戳列表]
```

#### 判断逻辑

1. 获取当前IP的历史失败次数（在阶梯窗口内的）
2. 查表得到封锁时间
3. 计算最后一次失败时间 + 封锁时间
4. 如果当前时间 < 解封时间，返回429 + 剩余秒数
5. 否则允许尝试

#### 响应示例（429）

```json
{
  "detail": "登录失败次数过多，请 45 秒后再试"
}
```

### 3. 前端UI

#### 设置页新增"安全设置"分区

在左侧导航新增"安全"按钮，点击切换到安全设置区域。

内容：
- 修改密码表单：旧密码、新密码、确认新密码、提交按钮
- 登录保护配置：最大尝试次数（默认10）、阶梯封锁时间表（只读展示）

#### 登录页改动

1. 增加"修改密码"链接（已登录时跳转设置页，未登录时提示先登录）
2. 底部增加折叠区域"忘记密码？"，点击展开显示：
   ```
   管理员强制重置密码：
   1. 删除 data/admin_auth.json 文件
   2. 重启服务
   3. 重新访问此页面设置新密码
   ```

### 4. 配置存储

#### settings.json 新增字段

```json
{
  "admin_max_attempts": 10,
  "admin_lockout_base_seconds": 60
}
```

- `admin_max_attempts`：每个阶梯的尝试次数（默认10）
- `admin_lockout_base_seconds`：第一阶梯封锁时间（默认60秒）

#### 阶梯计算逻辑（基于这两个参数）

阶梯系数固定，不可配置：

```
阶梯1: base_seconds * 1 = 60秒
阶梯2: base_seconds * 2 = 120秒
阶梯3: base_seconds * 4 = 240秒
阶梯4: base_seconds * 10 = 600秒
阶梯5: base_seconds * 60 = 3600秒
阶梯6: base_seconds * 1440 = 86400秒
```

每个阶梯包含 `admin_max_attempts` 次尝试机会。失败记录在 86400 秒（24小时）后自动过期清理。

#### admin_auth.json 不变

- 继续存储 `password_hash`、`revoked_sessions`
- 不存储封锁状态（内存即可）

| 文件 | 改动 |

| `admin_auth.py` | 新增修改密码函数 |
| `config.py` | 新增配置项 |
| `static/fragments/admin/settings.html` | 新增安全设置分区 |
| `static/admin-login.html` | 增加修改密码入口和忘记密码提示 |

## 测试要点

1. 修改密码：旧密码错误、新密码不一致、密码过短、成功修改
2. 修改密码后所有会话失效
3. IP封锁：连续失败触发封锁、封锁期间拒绝请求、封锁解除后允许登录
4. 阶梯递增：不同失败次数对应不同封锁时间
5. 设置页配置修改生效
