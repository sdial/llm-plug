# JSON 查看器 - 字段过滤功能设计

## 概述

为 `static/json-viewer.html` 增加字段级过滤能力，让用户可以从 JSON 树中选择性地查看特定字段。主要服务于 LLM 插件调试场景，数据来源包括 HTTP 头部（扁平）和 LLM API 请求/响应体（深层嵌套 JSON）。

## 需求

- 侧边栏树形字段面板，展示 JSON 所有字段路径
- 勾选控制字段显隐，未勾选字段从 JSON 中完全移除
- 支持 `?fields=` URL 参数指定默认勾选字段
- 深色主题，与现有风格一致
- 暂不实现搜索功能

## 整体布局

```
┌──────────────────────────────────────────────────────┐
│  JSON 查看器                              [收起侧栏] │
├──────────────┬───────────────────────────────────────┤
│ [全选][反选] │                                       │
│ ──────────── │                                       │
│ ▼ □ data     │  {                                    │
│     □ id     │    "data": {                          │
│     □ model  │      "model": "gpt-4",                │
│     ▼ □ ...  │      "choices": [...]                 │
│ ▼ □ usage    │    }                                  │
│     □ ...    │  }                                    │
│ ──────────── │                                       │
│ 已选 12/20   │                                       │
└──────────────┴───────────────────────────────────────┘
```

- 左侧树形面板固定宽度 280px，可通过拖拽调整
- 操作按钮（全选、反选）置于侧边栏顶部
- 底部状态栏显示 `已选 X/Y` 计数
- 右侧 JSON 主区域显示过滤后的结果

## 树形面板交互

### 展开/折叠

- 点击箭头 `▶` / `▼` 展开或折叠子节点
- 字段数 ≤ 50 时默认全部展开，> 50 时默认折叠

### 勾选逻辑（级联）

- 勾选父节点 → 所有子节点自动勾选
- 取消父节点 → 所有子节点自动取消
- 子节点部分勾选时 → 父节点显示半选状态（`─`）
- 所有子节点勾选时 → 父节点自动勾选

### 操作按钮

- **全选**：勾选所有字段
- **反选**：反转所有字段的勾选状态

## JSON 显示区行为

### 过滤规则

- 取消勾选的字段从 JSON 树中完全移除（不保留占位）
- 如果父对象的所有子字段都被取消，则整个父对象也被移除
- 数组元素按索引作为字段处理，可单独勾选
- 空对象 `{}` 和空数组 `[]` 不显示

### 过滤示例

原始 JSON：
```json
{
  "data": {
    "model": "gpt-4",
    "choices": [{"text": "hello"}],
    "usage": {"total": 100}
  },
  "status": "ok"
}
```

只勾选 `data.model` 和 `data.choices` 时显示：
```json
{
  "data": {
    "model": "gpt-4",
    "choices": [
      {
        "text": "hello"
      }
    ]
  }
}
```

### JSON 区域折叠

- `{}` / `[]` 可点击折叠（复用现有能力）
- 折叠状态下显示字段数量，如 `{3 fields}`

## 数据来源

### 现有参数

```
?url=<json-api-url>&title=<page-title>
```

### 新增参数

```
?fields=data.model,data.choices
```

- 不传 `fields`：默认全部勾选（向后兼容）
- 传 `fields`：只勾选指定路径的字段，其余隐藏
- 路径用 `.` 分隔，如 `data.choices.0.text`
