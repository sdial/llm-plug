# 渠道管理页面优化设计

## 目标

将渠道管理页面的列表从"表格+抽屉"模式优化为简洁的单行展示，提升操作效率和信息密度。

## 当前问题

1. 每个渠道需要展开抽屉才能查看详情（模型列表、参数等）
2. 操作按钮分散，删除按钮直接暴露容易误操作
3. 状态切换需要单独的"禁用/启用"按钮
4. API类型显示过长（openai-chat-completions 等）
5. Base URL 占用空间但不是核心信息

## 优化方案

### 1. 单行布局

去掉抽屉，所有关键信息集中在一行展示：

| 列名 | 宽度 | 内容 |
|------|------|------|
| 名称 | 140px | 渠道名称 |
| 状态 | 90px | 启用/禁用徽章（可点击切换） |
| 类型 | 60px | C/R/A 单字母徽章 |
| 模型 | 220px | 模型列表，完整展示 |
| Base URL | 200px | API地址，完整展示 |
| 操作 | 120px | 编辑 + 测试按钮 |

### 2. 状态列交互

- 显示启用/禁用状态徽章
- 点击徽章触发 POP 确认弹窗
- 确认后切换状态
- 视觉样式：
  - 启用：绿色背景 + ✓ 图标
  - 禁用：灰色背景 + ✗ 图标

### 3. API类型简写

用单字母徽章替代完整类型名：

| 类型 | 简写 | 颜色 |
|------|------|------|
| openai-chat-completions | C | 紫色 #635bff |
| openai-response | R | 蓝色 #2563eb |
| anthropic | A | 橙色 #d97706 |

hover 时显示完整类型名称。

### 4. 操作按钮简化

- 只保留"编辑"和"测试"两个按钮
- "删除"按钮移入编辑弹窗中
- 状态切换通过状态列徽章完成

### 5. 测试功能

- 点击"测试"按钮弹出模型选择弹窗
- 选择模型后执行测试
- 显示测试结果（成功/失败、延迟、回复）

## 交互流程

### 状态切换
1. 用户点击状态徽章
2. 弹出确认弹窗："确定要禁用/启用该渠道吗？"
3. 用户确认后切换状态
4. 刷新列表

### 测试连接
1. 用户点击"测试"按钮
2. 弹出模型选择弹窗（下拉框 + 确认按钮）
3. 选择模型后执行测试
4. 显示测试结果

### 编辑渠道
1. 用户点击"编辑"按钮
2. 打开编辑弹窗（包含删除按钮）
3. 可修改所有字段或删除渠道

## 技术实现

### 涉及文件
- `static/index.html` - 主要修改文件

### 修改点

1. **表格结构调整**
   - 删除抽屉相关的 HTML 结构
   - 调整表头和列宽
   - 移除展开/折叠图标列

2. **renderChannels 函数重写**
   - 简化为单行渲染
   - 状态徽章添加点击事件
   - API类型映射为简写

3. **新增交互函数**
   - `toggleStatusWithConfirm(id)` - 带确认的状态切换
   - `openTestModal(id)` - 打开测试模型选择弹窗
   - `executeTest(id, model)` - 执行测试

4. **新增弹窗**
   - 确认弹窗（复用现有样式）
   - 测试模型选择弹窗

5. **编辑弹窗修改**
   - 添加删除按钮

## Why

用户更关注模型列表和测试功能，当前抽屉设计增加了操作步骤。单行布局让信息一目了然，减少点击次数。

## How to apply

此优化仅影响渠道管理 tab，不影响其他 tab 的功能。
